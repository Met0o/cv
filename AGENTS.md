# AGENTS.md

## Project overview

This repository builds Metodi Simeonov's CV as a Jekyll site. The deployed site uses the `/cv` base URL.

## Source of truth

- Edit CV content in `_data/data.yml`.
- Edit section markup in `_includes/` and page composition in `_layouts/` or the root HTML files.
- Edit styling in `_sass/` and `assets/css/main.scss`.
- Do not hand-edit `_site/`; it is generated by Jekyll.
- Preserve unrelated user changes in the working tree.

## Local development

Install the locked dependencies:

```bash
bundle install
```

Run the local builder on the project development port:

```bash
bundle exec jekyll serve --host 0.0.0.0 --port 4001 --livereload-port 35730
```

Open <http://localhost:4001/cv/>. Section anchors can be tested directly, for example <http://localhost:4001/cv/#projects>.

In WSL, if `bundle` resolves to the Ruby 4 Snap and reports incompatible gems, invoke Bundler with the system Ruby 3.2 installation:

```bash
/usr/bin/ruby ~/.gem/gems/bundler-2.3.19/exe/bundle install
/usr/bin/ruby ~/.gem/gems/bundler-2.3.19/exe/bundle exec jekyll serve --host 0.0.0.0 --port 4001 --livereload-port 35730
```

## Validation

- Run `bundle exec jekyll build` after content, template, or style changes.
- Check the affected section at the `/cv/` URL and verify desktop and narrow/mobile layouts when presentation changes.
- Keep YAML indentation valid; `_data/data.yml` formatting errors can prevent the entire site from building.
- For project-section changes, verify that `http://localhost:4001/cv/#projects` scrolls to the rendered section.
